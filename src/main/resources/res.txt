Локомотив состоит из нескольких множеств, которые взаимосвязаны между собой и своими компонентами, которые постоянно нужно поддерживать в рабочем сосотоянии, для безопасности пассажиров и груза, но и прилегающей к путям инфраструктуры.
На данный момент есть несколько систем, которые используются на железной дороге, которые осуществляют запись на электронные носители текущих показаний датчиков локомотива.
Однако главным недостатком этих систем является их разрозненность и отсутствие программного обеспечения, позволяющего визуализировать и провести анализ накопленных данных.
Система интеллектуального диагностирования локомотивов предназначена для сбора всех данных в единое хранилище и осуществление их последующего анализа.
Конечным результатом работы системы является визуализация собранных данных, оценка текущего состояния и предупреждения пользователя о возможных неисправностях.
Чтобы импортировать данные с электронного носителя локомотива в хранилище системы, сотрудник железной дороги, во время остановки состава в депо, может воспользоваться web-интерфейсом, указав необходимый файл, содержащий собранные данные.
Для решения проблемы несовместимости форматов систем, используемых на локомотивах для записи данных с приборов, применяется специальный модуль, который превращает их в единый формат, после чего они заносятся в хранилище.
Поскольку количество обрабатываемых системой данных превышает допустимый объем реляционных баз данных, для реализации хранилища используются технологии, поддерживают высокую степень масштабируемости и предназначены для хранения больших объемов данных.
Применение данных технологий позволяет значительно увеличить производительность системы за счет запуска ее на кластере из нескольких компьютеров.
